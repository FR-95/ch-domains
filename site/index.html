          Methodology

          • The .ch zone snapshot is refreshed daily by a GitHub Action.
          • Per-day changes are computed from git diffs of consecutive sorted snapshots.
          • Added lines correspond to registrations.
          • Removed lines correspond to deregistrations.
          • Patterns, records, churn and 7-day averages are derived from this daily series; the initial backfill day is excluded so it cannot distort averages or peaks.
          • Composition stats analyse the label before .ch. IDN domains are stored as punycode (xn--…) and decoded for display. Whole-zone figures are precomputed at build time; per-day figures are computed in the browser.
          • The dashboard is fully static and reads only precomputed JSON files — there is no backend.
          Source data: this repository's git history · SWITCH .ch zone
